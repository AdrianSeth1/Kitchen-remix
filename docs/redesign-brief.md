# Redesigning Kitchen Remix with Claude Design

Goal: a refined **premium dark** look, without breaking the working app.

Claude Design (claude.ai/design) builds a polished *prototype*. Your real frontend
has live API wiring and the data flow that was just repaired. So the safe path is:
**design the look there, then bring it back as styling only** — never let it
regenerate and re-wire your actual components.

## Step by step

1. **Access.** You need Claude Pro/Max/Team/Enterprise. Go to **claude.ai/design** (research preview).

2. **Give it context.** Easiest start:
   - Run the app (see `QUICKSTART` / README), screenshot the main screen, and **upload the screenshot** so it restyles what exists.
   - Also paste `frontend/src/App.jsx` and one panel (e.g. `FinishSelector.jsx`) so it learns the component structure and Tailwind approach.
   - Optional: point it at the repo so it can extract the current palette/components into a design system.

3. **Prompt it** with the brief below.

4. **Refine on the canvas.** Use inline comments and the sliders it generates
   (accent intensity, corner radius, density) until it looks right. Iterate in
   conversation — that's the whole point of the tool.

5. **Export what you need — tokens, not a new app.** Ask it specifically for:
   - a **Tailwind theme** (the `theme.extend` block: colors, fontFamily, borderRadius, boxShadow), and
   - the **restyled `className` strings** for each component (or a small set of reusable class combos).

6. **Bring it back safely.** Hand those tokens + classes to your coding model
   (Claude Code with the **frontend-design plugin** is built for exactly this) with
   the constraint below. Branch first: `git checkout -b redesign`.

## Paste this into Claude Design

```
Restyle an existing React + Vite + Tailwind web app called "Kitchen Remix" — a
tool where someone uploads a photo of their kitchen and previews remodel options
(new cabinets, countertops, paint, etc.) generated by an AI image model. Audience:
the owner's parents (non-technical) AND recruiters reviewing it as a portfolio
project. So it must feel polished and trustworthy, not gimmicky.

Direction: PREMIUM DARK — refined, designer-grade, calm. Not a flat dark theme.
- Layered near-blacks for depth: a base background, slightly raised panels, and
  raised controls — each a step lighter — with hairline borders.
- ONE warm accent color used sparingly (copper / warm amber feels right for a
  kitchen — it echoes brass and wood finishes). Use it only for primary actions,
  active states, and focus rings. Keep a subtle success-green for the "backend ok"
  status chip.
- A clear type scale with a strong display treatment for headings and comfortable
  body text. Generous, consistent spacing rhythm. Rounded cards. Soft shadows and
  faint inner highlights for depth. Subtle hover/focus transitions.
- It should feel like a high-end design product, restrained and confident.

The app's panels (keep all of them, restyle each): a top photo-upload dropzone, a
header with a backend-status chip, Finish Presets (category tabs + selectable
tiles), A/B Comparison (results grid), Layered Editing (step list), Before/After,
Structural Experiments (inputs + caveats), Reference-based Edits (kitchen thumbnail
+ example upload + dropdown + free-text note), and an Export Spec Sheet button.

Deliverables I want from you:
1. A Tailwind theme.extend block (colors, fontFamily, borderRadius, boxShadow).
2. Reusable class combos for: page background, card/panel, primary button,
   secondary button, input, select, tab (active/inactive), result tile, caveat box.
3. The restyled version of the main screen so I can see it.

Do NOT change any app logic, component props, state, or fetch calls — this is a
visual restyle only. Output styling I can apply to existing components.
```

## The rule for bringing it back (give this to Claude Code)

> Apply the design tokens and class strings from Claude Design to the existing
> components. This is **visuals only**: edit `tailwind.config.js`, `index.css`, and
> `className` strings / JSX markup. Do **not** touch component props, `useState`,
> `fetch`/`/api` calls, or the data flow (`Upload → uploadedImageB64`,
> `FinishSelector → selectedFinishes → AblGrid → abResults → ExportSpecSheet`).
> Restyle every panel: Upload, header/status chip, FinishSelector, AblGrid,
> LayerEditor, BeforeAfterSlider, StructuralTab, ReferenceTab, ExportSpecSheet.
> Then run `npm run build` and confirm every panel still renders.

## Verify before merging

- `npm run build` passes.
- Upload a photo → each panel renders and is usable.
- The status chip, caveats, and reference-tab kitchen thumbnail are all still there.
- Then merge the `redesign` branch.
